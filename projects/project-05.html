Design System · 组件 · 2023

设计系统组件库

设计职责

主导 Design Token 定义、基础与业务组件库搭建、设计文档编写，推动设计与开发协作规范落地。

沉淀 120+ 可复用组件，设计到开发交付周期缩短 40%。

设计背景

多条产品线并行，各团队重复设计相似组件，视觉与交互不一致，设计与开发沟通成本高。

需求分析

  • 设计师：快速拼装页面，减少重复劳动
  • 开发：对齐组件 API，降低联调返工
  • 产品：保证跨产品体验一致性

设计思路

按原子设计方法论分层：Token → 基础组件 → 业务组件 → 页面模板，每个组件附带用法说明与禁忌示例。
